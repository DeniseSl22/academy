WikiPathways: QA additional tasks

Background

This step covers a few remaining quality assurance tasks that are part of the basic quality assurance protocol. If you have additional time after completing these, or have spent less than 30 minutes on the basic protocol, continue to additional quality assurance tasks by clicking the Next step button.

Instructions

Delete pathways tagged for deletion

The goal of this task is to removed pathways with expired deletion tags. Note that deletion can only be performed by sysop accounts

  • Follow the link to review the list of pathways tagged for deletion: Pathways tagged for deletion
  • Check the last column for pathways that are ready to be deleted; look for the delete icon.
  • Simply click on the delete icon to delete these pathways

Jenkins Test Report

The main goal of this task is to perform a few quick and easy pathway content curation tasks, for the Curated collection, which is routinely used in data analysis. The scope of this work should be limited to about 15-30 minutes.

  • Click the link below to see the Jenkins Test Report:
    Jenkins Test Report
  • Check the "All Failed Tests" list for any of the following test names:
    • nl.unimaas.bigcat.wikipathways.curator.General.noIdentifierURIs -- Which identifies datanodes that are annotated with something other than an identifier, e.g., a symbol or blank
    • nl.unimaas.bigcat.wikipathways.curator.EnsemblGenes.*** (e.g., wrongEnsemblIDForRatSpecies) -- Which identifies incorrect or unrecognized Ensembl IDs
      Note: You can ignore the other failed tests, which are better suited as topics for future Curation Events.
  • Click on the + to expand these test results, for example:

    • Data nodes with a 'noIdentifier' URI: [["node","label","homepage"], ["http://bio2rdf.org/geneid:noIdentifier", "TKT", "http://www.wikipathways.org/instance/WP421_r79853"] ] expected:<0> but was:<1>

    This means that on pathway WP421 the data node labeled TKT is missing a proper identifier.

    • For each error message, open the relevant pathway and update the problematic annotation
    • Double check to make sure the system name is correct
    • And don't forget to update curation tags after editing

    Test search

    The goal of this task is to make sure that the search feature is working and the index is up-to-date.

    • Search for the newest pathway by title by copying the pathway title from the top of the New Pathways page listing, but make sure the pathway is public and not private.
    • Paste the title into the search field in the left sidebar
    • There may be many results, but be sure that the newest pathway is among them. If not, then send an email to the WP dev list (wikipathways-devel AT googlegroups DOT com) describing the problem.

    Admin curation logistics

    • Send a curation report to the development team (wikipathways-devel AT googlegroups DOT com) summarizing your activities.
    • That's it! Thanks for helping to make WikiPathways a high-quality resource for pathway information!
