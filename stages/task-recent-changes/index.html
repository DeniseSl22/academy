WikiPathways: Assess Recently Changed Pathways

Background

The goal of this task is to assess recent pathway edits and newly added pathways, and to make sure that relevant tags are added and/or updated.

Instructions

  • For the pathway below, click the image or title to view/edit the pathway at WikiPathways. You will need to be logged in at WikiPathways in order to access the editing controls.
  • Perform the following tasks:

    • Review edit history

  • Scroll down to the History section and review the comments for recent changes. Compare to views of prior revisions.
  • If the recent edits are inaccurate or degrade quality, revert the pathway to a revision prior to the latest set of edits.
  • If you reverted an edit (or are uncertain), start a discussion on the pathway and/or with the author directly, explaining your reasoning.
  • If the pathway is new, also perform the following tasks:
  • Assess the technical and stylistic aspects of the content. Consider adding suggested improvements to your welcome message (see below) if you think it would be helpful to the author.

    • Fix bad titles

    • Pathway titles should be descriptive. To find a reasonable title, you may need to review the biology described in the pathway or the literature references indicated by the author.
    • Pathway titles should be unique. Check the Browse page to see if other pathways with the same title already exist. If so, compare the two pathways and make one or both of the titles more descriptive to highlight the difference.

      • Note: We encourage having several versions of the same pathway, for example Apoptosis, as long as each pathway describes a useful variation of the process, or is specialized for a specific tissue or state.

    Add ontology tags

    • If the pathway is new, add at least one top-level Pathway Ontology tag. For example, for a metabolic pathway, add the top-level tag "classic metabolic pathway".
    • If possible, also add any other appropriate ontology tags from either of the three categories; Pathway Ontology, Disease and Cell Type.

    Add quality tags

    • Tag the pathway as "Under construction" if it was created in the past 30 days and appears incomplete.
    • If the pathway looks like it was created as a test or tutorial, tag as "Test pathway". Many tutorial pathways will be based on the "circadian clock pathway" used in the Tutorial, or will contain just a few generic datanodes and connections. These will be tagged for deletion after 30 days. For details on how to add quality tags, see the Tutorial.

    Contact all new users

    • If the author is new to WikiPathways (check by clicking on username and "my contributions"), then be sure to start a discussion thread on their new pathway welcoming them and offering help:

      "Thanks for starting a new pathway! If you need any help with our editing tools, just let us know."

    • If the pathway looks suspiciously like a test pathway, then tag it as such, but still post a welcome message, perhaps adding:

      "If you are just testing the site, that's okay too. Just delete the pathway when you are done, or let us know and we'll take care of it."

      • Every new author should have a welcome message on their first pathway attempt.

    Identify pathways for social media

    • Any pathway that looks unique and complete should be highlighted through social media. Suggest that the author share his/her work via twitter, blogs, emails, etc. via your welcome message. Also contact Kristina Hanspers about tweeting the pathway via @wikipathways.

    Community approval tags

    • If a changed pathway is tagged with a community approval tag, e.g., "Reactome Approved", collect a summary of pathway page links and change details to send to each community representative. If you are not already in contact the representative, simply include the summary of links and details in your wrap-up email to wikipathways-devel and we'll take it from there.

    Add and update quality tags

    • Update faded tags if the recent edits are good by clicking on the click here link provided with "Approved version" and "Featured version" tags.
    • Consider adding new tags, for example, if the following criteria are met:
      • Approved version: Interactions must be complete, most datanodes must be annotated. About 80% coverage of Gene Product datanodes is sufficient, given that some elements might not have mappings. Overall, the pathway should be useful for downstream data visualization and analysis.
      • Featured version: In addition to the "Approved version" requirements, the pathway must have a good description and one or more ontology tags. Non-typical pathways, including gene lists and abstract networks, should be excluded. Overall, a Featured pathway should have a consistent and informative annotated layout that contributes to the understanding of the biological process.
      • For both Featured and Approved collections, DO NOT include list-based pathways and other pathways that don't describe a biological process.
    • Remove tags that have been addressed and are now outdated. For example the "Add description" tag, "Annotate nodes" tag and "Fix interactions" tag.
    • Depending on which tags were updated, added or removed for each pathway, click the corresponding "Verify" buttons below. For example, if a description was added and the "Approved version" tag was added, click the "Verify add description" and "Verify Approved version tag added". Note: The above criteria are used mainly for the archive of human and mouse pathways. For species other than human and mouse, these criteria are slightly more relaxed. Specifically, pathways don't need to have completed interactions, as long as most datanodes are annotated. Even without completed interactions, these pathways are useful for gene set analysis.
